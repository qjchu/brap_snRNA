# raw_data_processing

This directory contains R scripts to process raw snRNA‑seq data generated by **Cell Ranger** into a fully analysed **Seurat** object. The scripts handle data loading, quality filtering, normalisation, dimension reduction, clustering, and (for multi‑sample data) integration.

---

## Scripts

| Script | Description |
|--------|-------------|
| **`process_multi_samples.R`** | Loads two (or more) 10x samples, merges them, performs quality filtering, runs standard Seurat workflow, and applies three integration methods (CCA, RPCA, Harmony). Exports the integrated object, metadata, marker genes, and aggregated expression tables. |
| **`process_single_sample.R`** | Loads a single 10x sample, performs the same basic Seurat workflow (without integration), and exports the processed object, metadata, marker genes, and aggregated expression tables. |

> **Note**: The filenames above are placeholders. In practice, they are named `get_multi_samples_rds.R` and `get_single_sample_rds.R` (or similar). Adjust accordingly.

---

## Input Requirements

- **Cell Ranger output** for each sample, specifically the `outs/filtered_feature_bc_matrix` directory.
- **R (≥ 4.0)** with the following packages installed:
  - `Seurat` (≥ 5.0)
  - `dplyr`, `ggplot2`, `patchwork`, `svglite`
  - `reticulate` (with Python environment containing `leidenalg`)
- The scripts assume a specific folder structure. **You must edit the file paths** inside the scripts to point to your own data directories.

---

## Output Files

After running either script, the following files are generated:

| File | Description |
|------|-------------|
| `*_merge_res1.rds` (or `*_res1.rds`) | The final Seurat object, saved as an RDS file. |
| `*_metadata_res1.txt` | Cell‑level metadata (including UMAP/t‑SNE coordinates, cluster assignments, etc.). |
| `*_res1_cluster_markers.txt` | Marker genes for each cluster (positive markers only). |
| `*_RNA_origident_sum.txt` | Aggregated expression (sum) per gene across cells of each original sample. |
| `*_RNA_origident_avg.txt` | Average expression per gene across cells of each original sample. |
| `*_RNA_clusters_res1_sum.txt` | Aggregated expression (sum) per gene across cells of each cluster. |
| `*_RNA_clusters_res1_avg.txt` | Average expression per gene across cells of each cluster. |

For the multi‑sample script, additional embeddings and cluster assignments are saved for each integration method (CCA, RPCA, Harmony) inside the metadata file.

---

## How to Run

1. **Set working directory** – modify the `setwd()` command in the script to your analysis folder.
2. **Update file paths** – replace the `data.dir` arguments with the actual paths to your Cell Ranger output folders.
3. **Adjust Python path** – ensure the path to your Python executable (with `leidenalg`) is correct. Modify the `reticulate::use_python()` line if needed.
4. **Execute the script** in an R session:
   ```r
   source("process_multi_samples.R")   # or process_single_sample.R
   ```

---

## Notes

- The scripts use **Leiden clustering** (`algorithm = 4`). Make sure `leidenalg` is available in your Python environment.
- The resolution is set to `1`; you can change it if finer/coarser clusters are desired.
- For the multi‑sample script, integration is performed **after** the initial unintegrated workflow. The final identity is set to the Harmony clusters, but you can choose any integration result.
- All output files are written in tab‑separated text format for easy downstream use.

---

## Customisation

- To process **more than two samples**, extend the merge call accordingly.
- To use **other integration methods**, add them following the same pattern as CCA/RPCA/Harmony.
- To change the **filtering thresholds**, modify the quantile values in the `subset()` step.

---

## Dependencies

The scripts rely on the following R packages. Install them if missing:

```r
install.packages(c("dplyr", "patchwork", "ggplot2", "svglite", "reticulate"))
install.packages("Seurat")   # or BiocManager::install("Seurat")
```

For Python, install `leidenalg` via conda or pip:

```bash
conda install -c conda-forge leidenalg
# or
pip install leidenalg
```
